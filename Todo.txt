SECTION 1: CORE PHILOSOPHY & DESIGN THINKING
--------------------------------------------------------------------------------
1. The Assignment Goal: 
Build a minimal, highly polished web app where users can generate images from 
text prompts, view past generations, tweak old prompts, and crop images. 

2. The "Ship Small" Rule:
We are NOT building user authentication (login/signup) to save time. We are NOT 
building a complex backend server. We are using Next.js Server Actions/Routes 
and a serverless database (Supabase) to keep the app ultra-lightweight but 
incredibly robust.

3. The Unique Selling Proposition (USP) - Async Handling:
Image generation takes 2 to 5 seconds. Most developers will just put a boring 
spinning circle. TO STAND OUT, you will build a "Flawless Async UX":
- When generating, show a Skeleton Loader exactly the size of the expected image.
- Start a Live Digital Stopwatch (e.g., "Generating... 1.8s").
- If the AI API fails, gracefully stop the timer, remove the skeleton, and show 
  a beautiful Toast Notification with a "Retry" button. This proves you deeply 
  understand how to handle slow or failing APIs.

SECTION 3: DATABASE SCHEMA (SUPABASE)
--------------------------------------------------------------------------------
We need exactly ONE table in our Supabase database to store the Gallery.
Table Name: `generations`

Columns Required:
1. `id`         : Type UUID (Primary Key, Default: gen_random_uuid())
2. `created_at` : Type TIMESTAMP (Default: now(), used to sort gallery newest first)
3. `prompt`     : Type TEXT (The exact words the user typed)
4. `image_url`  : Type TEXT (The final image link provided by Fal.ai)
5. `settings`   : Type JSONB (Stores JSON data like: {"aspect_ratio": "16:9"})

Why Supabase? It allows us to fulfill the "Data Model" grading criteria perfectly 
without spending hours writing backend CRUD (Create, Read, Update, Delete) code.


--------------------------------------------------------------------------------
SECTION 4: GLOBAL STATE MANAGEMENT (ZUSTAND)
--------------------------------------------------------------------------------
File: `src/store/useStudioStore.ts`

The app needs a central "brain" so the Bottom Prompt Bar, the Right Gallery, and 
the Main Canvas can all talk to each other instantly.

STATE VARIABLES:
- `prompt` (string)       -> Binds to the input text box.
- `isGenerating` (boolean)-> Binds to the UI. If true, show the Skeleton + Timer.
- `currentImage` (string) -> The image currently displayed in the Main Canvas.
- `error` (string | null) -> Holds error messages if Fal.ai goes down.
- `history` (Array)       -> Holds the list of past generations from Supabase.

ACTIONS (FUNCTIONS):
- `setPrompt(text)`       -> Updates the text box as the user types.
- `generateImage()`       -> The powerhouse. Sets `isGenerating` to true, makes 
                             the API call, saves to DB, updates `currentImage`, 
                             adds it to `history`, and sets `isGenerating` to false.
- `loadHistory()`         -> Runs once when the page loads to fetch past DB items.
- `loadFromGallery(item)` -> Requirement #4. When a user clicks a Gallery thumbnail,
                             this instantly sets `prompt = item.prompt` and 
                             `currentImage = item.image_url`.


--------------------------------------------------------------------------------
SECTION 5: API INTEGRATION & DATA FLOW
--------------------------------------------------------------------------------
File: `src/app/api/generate/route.ts` (Next.js Backend API Route)

We CANNOT call Fal.ai directly from the frontend, otherwise our secret API keys 
will be exposed. The flow must be:

1. User clicks "Generate" -> Zustand calls `/api/generate`.
2. Next.js Server receives the `prompt`.
3. Next.js Server securely sends the prompt to Fal.ai using the FAL_KEY.
4. Fal.ai processes the image and returns a URL: `https://fal.media/image.jpg`.
5. Next.js Server takes that URL and the `prompt`, and INSERTS them into Supabase.
6. Next.js Server responds to the Frontend with { success: true, data: { image_url } }.
7. Frontend displays the image.

*CRITICAL ERROR HANDLING:* 
If Step 3 fails, the backend must return a 500 status code with `{ error: "Fal API Failed" }`. 
The frontend must catch this, stop the loading animation, and alert the user.

SECTION 7: BONUS FEATURE INSTRUCTION - CANVAS EDIT (CROP)
--------------------------------------------------------------------------------
Requirement: "Bonus: A basic canvas edit such as crop."

How to execute flawlessly:
1. Below the generated image in Zone D, place a "Crop" icon button.
2. When clicked, open a centered Modal overlay.
3. Use the `react-image-crop` library to display the `currentImage` inside the Modal.
4. Allow the user to drag a crop box.
5. Create a "Save Crop" button.
6. When clicked, use the native HTML5 Canvas API to draw the cropped region, 
   convert it to a Base64 URL (`canvas.toDataURL('image/jpeg')`), close the modal, 
   and replace the `currentImage` in Zustand with this new Base64 string.
